# 业务报表样式指南

> 与 `assets/template.xlsx` 一一对应。生成 Excel 时所有参数取自本表，**不要自行发明色号或字号**。

## 1. 调色板

| 用途             | 名称     | 十六进制   | 文字色   | 说明                           |
| ---------------- | -------- | ---------- | -------- | ------------------------------ |
| 标题行底色       | 深蓝     | `#293764`  | 白       | 大标题（如"2026 年 5 月销售月报"） |
| 表头底色         | 品牌蓝   | `#4A70AE`  | 白       | 列标题                         |
| 斑马纹偶数行     | 浅蓝     | `#F2F6FC`  | 黑       | 隔行底色，提高可读性           |
| 合计行底色       | 淡蓝     | `#D9E2F3`  | 黑       | 末行合计                       |
| 边框             | 浅灰     | `#BFBFBF`  | —        | 单元格四边                     |
| 表头描边         | 品牌蓝   | `#4A70AE`  | —        | 表头上下用 medium 加粗         |

> 文字色除标题/表头为白外，其余默认黑 (`#000000`)。

## 2. 字体

| 区域         | 中文字体     | 西文字体 | 字号 | 加粗 |
| ------------ | ------------ | -------- | ---- | ---- |
| 大标题       | 微软雅黑     | Arial    | 14   | 是   |
| 表头         | 微软雅黑     | Arial    | 11   | 是   |
| 正文         | 微软雅黑     | Arial    | 11   | 否   |
| 合计行       | 微软雅黑     | Arial    | 11   | 是   |

## 3. 单元格

| 属性     | 值                                       |
| -------- | ---------------------------------------- |
| 对齐     | 文字列 `left/center`，数字列 `right`     |
| 自动换行 | `wrap_text=True`（避免文字溢出列宽）     |
| 边框     | 正文：四边 `thin`；表头：上下 `medium`   |
| 数字格式 | 货币 `'"¥"#,##0.00'`；整数 `'#,##0'`     |
| 日期     | `'yyyy-mm-dd'`                           |
| 百分比   | `'0.00%'`                                |

## 4. 布局规则

1. **大标题**独占第 1 行，`merge_cells` 跨所有列；行高 32。
2. **表头**位于第 2 行；行高 24。
3. **数据行**从第 3 行开始，**斑马纹**（偶数行 `#F2F6FC`）。
4. **合计行**位于数据末尾，**单独一格写"合计"** 并合并到表头前的所有列。
5. **冻结** `freeze_panes = "A3"`，用户滚动时表头与标题始终可见。
6. **关闭网格线** `ws.sheet_view.showGridLines = False`，让边框和底色主导视觉。
7. **列宽**：根据表头文字与数据长度估算后用 `ws.column_dimensions[col].width` 设置；不要全部用默认 8.43。

## 5. 打印 / 导出

- 方向：横向（数据列多时）
- 缩放：`fitToWidth=1, fitToHeight=0` 配合 `fitToPage=True`
- 页边距：左右 0.5 英寸
- `print_options.horizontalCentered = True`

## 6. 不该做的事

- ❌ 同一份工作簿混用 3 种以上字体
- ❌ 用红色 / 黄色 / 绿色"装饰"——只在表达数据含义时使用（涨绿跌红须配合金融惯例）
- ❌ 把数字当文本写入（会被 `SUM` 跳过）
- ❌ 数字用左对齐（默认就是 `right`）
- ❌ 行高全部用默认 15
- ❌ 表头单元格留空
- ❌ 隐藏列后不告知用户（用户看不到但会算入合计，导致错愕）

## 7. 字段识别（典型业务报表）

如果用户没有明确列名，从内容推断常见列：

| 关键词                       | 列名建议           | 数字格式         |
| ---------------------------- | ------------------ | ---------------- |
| 金额 / 销售额 / 收入 / 成本 | 销售额(元)         | `'"¥"#,##0.00'`  |
| 数量 / 个数 / 件数 / 库存   | 销售量(件)         | `'#,##0'`        |
| 占比 / 比例 / 增长率         | 占比               | `'0.00%'`        |
| 日期 / 时间                 | 日期               | `'yyyy-mm-dd'`   |
| 部门 / 区域 / 销售员 / 客户 | 文字列（不格式化） | —                |
