# 商务汇报 PPT 样式指南

> 与 `assets/template.pptx` 一一对应。生成 PPT 时所有参数取自本表，**不要自行发明色号、字号或版式**。

## 1. 调色板

| 用途             | 名称     | 十六进制   | 适用场景                       |
| ---------------- | -------- | ---------- | ------------------------------ |
| 主色（深蓝）     | Deep Blue| `#293764`  | 封面背景、章节分隔页、强调文字 |
| 品牌蓝           | Brand    | `#4A70AE`  | 顶部色条、装饰条、强调短线     |
| 辅蓝             | Accent   | `#6E9FCB`  | 副标题、装饰文字               |
| 浅底（卡片底）   | Light BG | `#F2F6FC`  | KPI 卡片底、表格偶行底         |
| 正文文字         | Dark     | `#1F1F1F`  | 正文                           |
| 副文文字         | Gray     | `#808080`  | 引言、页脚                     |
| 反白             | White    | `#FFFFFF`  | 深底上的文字                   |

## 2. 字体

| 区域         | 中文字体 | 西文字体  | 字号  | 加粗 |
| ------------ | -------- | --------- | ----- | ---- |
| 封面主标题   | 微软雅黑 | Arial     | 44    | 是   |
| 封面副标题   | 微软雅黑 | Arial     | 36    | 是   |
| 章节大数字   | —        | Arial     | 140   | 是   |
| 章节名       | 微软雅黑 | Arial     | 36    | 是   |
| 内容页 H1    | 微软雅黑 | Arial     | 24    | 是   |
| KPI 数值     | —        | Arial     | 28    | 是   |
| 正文         | 微软雅黑 | Arial     | 13–18 | 否   |
| 页脚         | 微软雅黑 | Arial     | 9     | 否   |

中英文混排时**分别设置字体**：中文用 `微软雅黑`，数字与英文用 `Arial`。在 python-pptx 中通过 `run.font.name` 设置（注意：python-pptx 会同时作用于中文与西文，如需精细控制可设置 `<a:ea>` 东亚字体与 `<a:latin>` 字体，本 skill 不强求）。

## 3. 页面尺寸

统一使用 **16:9 宽屏**：
```python
prs.slide_width  = Inches(13.333)
prs.slide_height = Inches(7.5)
```
**不要**用 4:3（10×7.5 英寸），老旧且与现有样例不一致。

## 4. 六种标准页型

| #  | 页型        | 用途                          | 关键元素                              |
| -- | ----------- | ----------------------------- | ------------------------------------- |
| 1  | 封面        | 报告标题、汇报人、日期        | 左侧 5 英寸深蓝大色块 + 装饰细条       |
| 2  | 目录        | 4–6 章节列表                  | 编号 + 章节名 + 页码 + 分割线          |
| 3  | 章节分隔页  | 切换章节时用                  | 全屏深蓝 + 140pt 大数字 + 章节名       |
| 4  | 内容页（KPI）| 关键指标 + 解读                | 顶部色条 + 3 个 KPI 卡片 + 关键解读    |
| 5  | 内容页（表格）| 数据对比                      | 顶部色条 + 表头深蓝 + 斑马纹表格       |
| 6  | 结束页      | Q & A、联系信息                | 全屏深蓝 + "谢谢聆听" + 联系方式      |

**完整汇报最少 6 页**：1 封面 + 1 目录 + ≥3 内容 + 1 结束。**章节数 ≥ 4** 时，每个章节前插一页"章节分隔页"。

## 5. 内容页通用结构

每张内容页（页型 4 / 5）必须遵循：

1. **顶部色条**：宽 13.333 英寸 × 高 0.18 英寸，品牌蓝 `#4A70AE`
2. **标题**：左边距 0.6 英寸、距顶部 0.45 英寸，24pt 加粗深蓝
3. **引言**（可选）：14pt 灰色，1–2 句话概括
4. **主体内容**：从 y ≈ 2.1 英寸开始
5. **页脚**：汇报主题（9pt 灰色左对齐） + 页码（9pt 灰色右对齐）

## 6. 对齐与边距

- 整体边距：左右各 0.6 英寸
- 文本框之间：垂直间距 0.5 英寸为标准单位
- 段落：单倍行距，避免使用 1.5 倍
- 标题与正文：垂直间距 ≥ 0.4 英寸
- 项目符号：自定义 `add_filled_rect` 蓝方块比默认 `•` 更整齐

## 7. 表格规范

- 表头：深蓝底 `#293764` + 白字 + 13pt 加粗
- 奇数行：白底
- 偶数行：浅蓝底 `#F2F6FC`
- 行高：0.5 英寸
- 单元格内边距：左右各 0.2 英寸
- 文字列：左对齐；数字列：居中或右对齐（按数字惯例）
- **不画表格边框线**——背景色与对齐已足够，线条反而割裂视觉

## 8. 装饰元素（克制使用）

- **细色条**：宽 0.6–1 英寸 × 高 0.04 英寸，品牌蓝，用于分隔章节名与副名
- **左侧色块**：宽 0.04 英寸 × 整行高，品牌蓝，用于强调 KPI 卡片边界
- **大数字**：仅用于章节分隔页（140pt），不要用在内容页
- ❌ **不要**用：渐变、阴影、3D 形状、艺术字、低多边形背景、emoji 装饰

## 9. 数字格式

| 类型     | 格式            | 示例           |
| -------- | --------------- | -------------- |
| 货币     | `"¥"#,##0` 或 `"¥"#,##0.00` | ¥1,280 万 |
| 百分比   | `0.0%` 或 `0.00%`           | 12.4%   |
| 千分位   | `#,##0`                    | 1,847   |
| 同比变化 | `+12.4% YoY`               | —        |
| 百分点   | `+4.1pp`                   | 复购率变化 |

> 文本里写 `+12.4% YoY` 即可，**不要试图用代码格式化后塞进标题**——容易与字体冲突。

## 10. 不该做的事

- ❌ 同一份 PPT 用 3 种以上字体
- ❌ 把全文塞进一张幻灯片（> 30 行就要拆页）
- ❌ 表格行列数过多（> 8 列 → 拆成多张；> 12 行 → 改用图表）
- ❌ 章节分隔页 1 章节配 1 张（章节数 < 4 时不要硬加分隔页）
- ❌ 用红黄绿装饰"好看"
- ❌ 标题用纯黑 `#000000`（用深灰 `#1F1F1F` 视觉更柔和）
- ❌ 把 AI 生成的图（带水印或乱码）直接放进 PPT

## 11. 常见错误清单

| 现象                            | 原因                              | 解决                                  |
| ------------------------------- | --------------------------------- | ------------------------------------- |
| 中文显示为方块                  | 没装中文字体或字体名拼错          | 用 `"微软雅黑"`，并确认系统已装        |
| 封面副标题与主标题重叠          | top 偏移没拉开                    | 主标题 y=2.6"，副标题 y=3.6"          |
| KPI 数值在卡片里偏上/偏下        | 文本框 anchor 用错                | 用 `MSO_ANCHOR.MIDDLE` 或手动算 y     |
| 表格列宽不够，文字被截断        | 列宽按表头长度算，忽略数据长度     | 按 max(表头, 数据) × 0.18 英寸估     |
| 章节分隔页大数字盖住章节名      | 数字 y 偏移没留够                 | 数字 y=1.5"，章节名 y=4.0"           |
| python-pptx 报 `Inches(0.18)` 不准 | 用了 Emu 整数比较                 | 色条/线条用 `Emu(...)` 显式构造        |
