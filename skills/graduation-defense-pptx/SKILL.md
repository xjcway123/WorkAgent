# 毕业答辩 PPT 撰写

根据用户提供的论文、研究材料或毕业设计内容，生成一份结构完整、设计专业的毕业答辩汇报 PPT。本 skill 内置一套学术风格模板（蓝色 3D 立方体 + 校徽，南通大学风格），通过**模板编辑工作流**复用模板的版式与视觉设计，把内容改写填充进去。

## 前置：必读 pptx 技能

本 skill 依赖 `/mnt/skills/public/pptx/` 的脚本与工作流。**开始前先读 `/mnt/skills/public/pptx/editing.md`**，掌握 unpack → 增删/复制幻灯片 → 编辑文本 → clean → pack 的完整流程。所有 `scripts/` 路径都指向 `/mnt/skills/public/pptx/scripts/`。

## 模板位置

- 模板文件：本 skill 目录下 `assets/template.pptx`
- 模板详细解剖（每页类型、版式、设计参数、内容页档案、常见改法）：`references/template-anatomy.md` —— **规划与编辑阶段务必阅读**。

## 第一步：收集与梳理材料

答辩 PPT 的质量取决于内容组织。先从用户提供的材料（论文、摘要、实验结果、毕设报告等）中提取以下信息。**有缺失且无法从材料推断的关键信息（尤其是答辩人、导师、题目），用一次 `ask_user_input_v0` 一并询问，不要逐条追问。**

必备信息：
- **题目**：论文/课题完整标题
- **答辩人**姓名
- **指导老师**姓名（可多位）
- **学院 / 实验室 / 学校**（默认沿用模板的“南通大学 人工智能与计算机学院 智能软件工程实验室”，用户另有说明则替换）

内容信息（尽量从材料中归纳，组织成下面的标准章节）：
- **研究背景**：问题背景与意义、国内外研究现状、相关工作的不足
- **研究内容**：总体研究内容/目标、数据集、方法框架（可多个方法）、实验设置与结果、主要贡献与创新
- **系统展示**（可选）：若有系统/工具/Demo
- **研究成果**：发表论文、专利、竞赛获奖、荣誉奖学金等
- **致谢**：感谢对象

## 第二步：规划幻灯片映射

先看 `references/template-anatomy.md` 了解每个内容页档案（archetype）。然后据材料拟定页面清单。答辩 PPT 的**标准骨架**（按需增删章节与内容页）：

1. **封面**（branded 版式）
2. **目录**（branded 版式）——列出本次答辩的章节
3. 每个章节：**章节分隔页**（branded，大号“01”+章节名）+ 若干**内容页**（content 版式）
   - 研究背景：背景意义 → 研究现状 → 相关工作不足
   - 研究内容：研究内容总览 → 数据集 → 方法框架（每个方法一页）→ 实验设置与结果 → 贡献创新
   - 系统展示（可选）
   - 研究成果：论文 / 专利 / 竞赛 / 奖项
   - 致谢
4. **结束页**（branded，“感谢各位老师！”）

规划原则：
- **章节编号要连续**（01、02、03…）。模板原件致谢页误写成“06”，生成时改为正确连续编号。
- **目录页的条目必须与实际章节一一对应**。
- **版式多样化**：内容页要按内容类型选不同档案（表格页放数据集/实验结果，要点页放背景/不足，框架图页放方法，并列卡片页放贡献创新，清单页放成果）。不要每页都用同一种纯标题+要点的排版。详见 anatomy 文档的“内容页档案”。
- 答辩 PPT 篇幅通常 18–28 页；硕士答辩偏多，本科偏少。控制总页数，避免冗长。

## 第三步：构建演示文稿（结构改动）

1. 复制模板到工作目录：`cp assets/template.pptx /home/claude/work.pptx`
2. unpack：`python /mnt/skills/public/pptx/scripts/office/unpack.py /home/claude/work.pptx unpacked/`
3. 按映射方案**先完成所有增删与复制**（用 `add_slide.py` 复制要复用的页；删页要从 `presentation.xml` 的 `<p:sldIdLst>` 移除对应 `<p:sldId>`）：
   - 保留并复用：封面、目录、各章节分隔页、结束页（这些是 branded 版式，立方体与校徽来自版式，复制后自动带装饰）。
   - 内容页：从模板中选与新内容类型匹配的页复制，删掉用不到的原论文专属内容页。
   - 章节分隔页需要几个就从模板的分隔页（slide3/7/18/20/23 任一）复制几个。
4. 在 `<p:sldIdLst>` 中排好最终顺序。

## 第四步：编辑每页文本

逐页打开 `unpacked/ppt/slides/slideN.xml`，用 **str_replace（Edit）工具**把占位/原内容替换成新内容。遵循 `/mnt/skills/public/pptx/editing.md` 的格式规则与 `references/template-anatomy.md` 的逐档案改法。要点：

- **封面**：替换题目、答辩人、指导老师、学院/实验室。
- **章节分隔页**：替换大号数字（“01”等，128pt）和章节名（43pt）。务必同时改 `<a:r>` 里的可见文本和后面 `<a:endParaRPr>`（若有）。
- **结束页**：保留“感谢各位老师！”或按需替换。
- **内容页**：标题用 `b="1"` 加粗，蓝色 `4A70AE`；正文用 `微软雅黑`；数字/英文用 `Arial`。多条目用多个 `<a:p>`，不要拼成一段。
- **图表/截图**：模板里的论文专属配图、表格数据是原论文的，不适用于新课题。
  - 若用户提供了对应图片/截图/数据，替换进去（图片走 media 重新关联，表格逐格改文本）。
  - 若没有，**删除该图占位形状**并改用文字/要点表达，或在该处放一个明显的中文占位提示（如“〔此处插入方法框架图〕”）让用户后续补充——不要保留原论文的无关配图。
  - ⚠️ **图片型残留（最易漏！）**：柱状图、模型结构图、系统截图等常是**位图**，其中的旧文字（别的模型名、别的指标）被“烤”进像素里，`grep` 文本**查不到**，只能靠视觉质检肉眼发现。凡 `<p:pic>` 引用的较大位图（按 `<a:ext>` 面积筛 >2e12 EMU²）默认都视为原论文专属图，按上条删除并占位。删 `<p:pic>` 后同时删 `slideN.xml.rels` 里对应 `<Relationship>`，再跑 `clean.py` 清孤立 media。页签旁的小图标面积很小、通常通用，可保留。
- **整页重建时**：某页图形太多无法逐一替换（蒙太奇背景页、复杂流程图页），可整页重写 `spTree`（保留 a/r/p 命名空间头与 `<p:clrMapOvr><a:masterClrMapping/></p:clrMapOvr>`，装饰交给版式）。**重写后标题别顶到左上角**——内容版式 slideLayout1 左上角有圆形校徽，标题框放在 `<a:off x="1329690" y="1013460"/>` 一带（与模板原内容页标题对齐），否则首字被校徽压住；卡片/正文从 `y≈1750000` 起，避免与标题重叠。
- **配色规范**（主题色，可用 `schemeClr` 引用，也可直接写 srgb）：
  - 深蓝 `293764`（大数字、深色文字）
  - 品牌蓝 `4A70AE`（标题、强调）
  - 背景白 + 立方体装饰（来自版式，勿动）
- 引用文献：保留参考文献的脚注小字体风格；若新内容有参考文献按同样格式列出。

## 第五步：清理、打包、质检

1. **清空备注页**：模板的 `ppt/notesSlides/notesSlide*.xml` 常残留原答辩人的私人讲稿备注（如“背景少讲”“英文写成中文”等），务必清空——把所有 `<a:t>…</a:t>` 改成空 `<a:t></a:t>`（保留结构、校验合法即可），否则会随成品泄露给用户。
2. 清理：`python /mnt/skills/public/pptx/scripts/clean.py unpacked/`（删除已移除页、孤立 media/tags/notes）。重建过页的话先精简其 `.rels`（只留 slideLayout + notesSlide 关系），再 clean，可大幅减小体积。
3. 打包：`python /mnt/skills/public/pptx/scripts/office/pack.py unpacked/ /home/claude/output.pptx --original /home/claude/work.pptx`
4. **内容质检（文本残留）**：本环境无 `extract-text` 命令，直接对解包目录 grep 最可靠：
   ```bash
   grep -rinoE "原答辩人名|原课题关键词|GRACE|CWE|lorem|ipsum|TODO|\[insert" unpacked/ppt/slides/*.xml
   grep -rilE "原答辩人名|原课题关键词" unpacked/ppt/notesSlides/*.xml
   ```
   逐页编辑时就用 `assert 出现次数` 兜底；最后整体复查一遍，确认无原论文专属文字、章节编号连续、目录与章节一致。
5. **视觉质检（最关键）**：转图逐页看（命令见 pptx SKILL.md 的 Converting to Images；可 `montage` 拼 5×N 网格快速总览，再对可疑页 100dpi 单独放大）。**必查**：
   - **图片型残留**——柱状图/结构图/截图里“烤死”的旧文字（grep 查不到，只能看图，见第四步⚠️）；
   - 标题是否被左上角校徽压住（尤其整页重建的页）；
   - 文字溢出卡片/表格、立方体背景被遮挡、章节大数字与标题错位、表格变形、空占位框是否还在。
   发现问题修一轮即可，不要无限微调。
6. 输出：把成品复制到 `/mnt/user-data/outputs/`，用 `present_files` 呈现，文件名体现答辩人或题目（如 `毕业答辩_张三.pptx`）。

## 注意事项

- **保真优先**：尽量保留模板的版式、立方体背景、校徽、配色、字体；这是模板的灵魂。换学校时同时替换 `image5.png`（校徽）与封面/致谢的校名文字。
- **删图不删空壳**：当新内容比模板少元素时，整体删除多余的图片/形状/文本框，不要只清空文字留下空白图框（见 editing.md 的 Template Adaptation）。
- **中文排版**：中文用 `微软雅黑`，注意 `lang="zh-CN" altLang="en-US"`；含引号用 XML 实体（见 editing.md）。
- **务实**：答辩 PPT 重在清晰传达研究脉络与成果，不堆砌。每章节聚焦“做了什么、怎么做、效果如何”。
